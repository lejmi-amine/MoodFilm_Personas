User Scenarios

How users interact with MoodFilm

"Late-night Chill" (Emma)

Emma wants to find a relaxing movie before bed without spending 20 minutes scrolling.

Tasks:

  • Opens MoodFilm
  • Chooses "Chill & Relax" theme
  • Browses suggested movies
  • Picks one and starts watching

Challenges:

Too many similar options on other platforms; wants emotional match.

Solution:

MoodFilm filters by emotional tone → soft colors, relaxing soundtrack previews, AI-curated "calm" picks.

"Family Movie Night" (Sophie)

Sophie wants to find a family-friendly movie that fits everyone's mood.

Tasks:

  • Opens MoodFilm
  • Selects "Family Night" theme
  • Filters by "All ages"
  • Checks movie descriptions and ratings

Challenges:

Balancing interests of kids and adults.

Solution:

Curated playlists for family nights, sorted by humor, animation, and adventure.

"After a Hard Day at Work" (Max)

Max wants a movie that matches his mood after a long day.

Tasks:

  • Opens MoodFilm
  • Selects "Tired" or "Motivation & Growth" mood
  • Gets recommendations with uplifting tone

Challenges:

Wants something meaningful but not too heavy.

Solution:

Personalized suggestions based on positive IMDb sentiment analysis or motivational keywords.

User Stories & Requirements

Defining what we need to build

User Story 1

As a movie lover, I want to choose movies based on my mood, so that I can enjoy a film that matches my emotions.

Acceptance criteria:

  • User can select a mood (e.g., happy, sad, excited, romantic)
  • Movies are recommended according to the selected mood
  • User can start watching directly from the recommendation

User Story 2

As a premium user, I want access to special themes like Girls' Night In or Family Night, so that I can easily find fitting movies for specific occasions.

Acceptance criteria:

  • Premium users see exclusive theme filters
  • Each theme provides a curated list of relevant movies
  • Non-premium users see a "locked" icon prompting upgrade

User Story 3

As a time-constrained user, I want quick and accurate movie recommendations, so that I don't waste time scrolling through endless options.

Acceptance criteria:

  • The system loads recommendations within 3 seconds
  • The top 5 movies always match the selected mood
  • User satisfaction feedback can be submitted ("Good match / Bad match")

User Story 4

As a casual viewer, I want to save my favorite moods and movies, so that I can re-watch or get similar recommendations later.

Acceptance criteria:

  • User can bookmark moods and movies
  • Favorites are stored in the user profile
  • Bookmarked movies influence future recommendations

User Story 5

As a parent, I want to apply family-safe filters, so that I can avoid inappropriate content during family nights.

Acceptance criteria:

  • Family-friendly filter hides adult-rated content
  • Parental control PIN available
  • Family Night mode defaults to safe selections

✅ Definition of Done (DoD)

  • All planned features (mood-based recommendation, premium themes, favorites, parental filter) are fully implemented and tested
  • Frontend and backend are successfully integrated and stable
  • All critical bugs are resolved and non-critical issues documented
  • Performance goals are met (e.g., recommendation loads < 3 seconds)
  • Security and privacy standards (API key handling, user data) are ensured
  • UI/UX has been reviewed and validated through user testing
  • Documentation for setup, usage, and maintenance is complete
  • The product has been deployed to a public or demo environment
  • Final acceptance testing with stakeholders (teacher / users) is completed

🟢 Definition of Ready (DoR)

  • Personas and scenarios for the story have been validated by the team
  • The story’s acceptance criteria are clearly defined and testable
  • Effort estimation (story points or time) has been completed
  • API sources and endpoints are identified and accessible
  • Dependencies or blockers have been clarified
